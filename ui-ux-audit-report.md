# UI/UX Audit Report — cv.sarkhan.dev

**URL audited:** https://cv-sarkhan-dev.vercel.app/workspace (plus /pricing, /telegram, /)
**Date:** 2026-07-06
**Viewport:** Desktop 1280×577 + responsive checks via code review
**Overall Score:** 5.5/10

---

## Executive Summary

The app has a polished dark/glassmorphic visual shell, consistent gradients and decent typography, but the core user journey in the workspace is currently broken by backend errors (HTTP 500). Several components are visually present but non-functional or misleading: the ATS score is always 0%, the pulse ring never renders, the Pro paywall is duplicated, the drag handle has almost zero affordance, and Material Symbols icons are not loaded so icons appear as text labels ("smart_toy", "send", "badge"). The Telegram page and Pricing page are coherent but the landing page is extremely bare and offers no value proposition.

## Component-by-Component Findings

### 1. Header
- **Status:** Functional but visually heavy.
- **Good:** Sticky, uses glass-card style, logo + brand readable, responsive Sign In icon on mobile.
- **Problem:** `font-geist` class is present but the design system mixes Geist + Inter inconsistently; no navigation back to workspace from landing page, no user avatar/logged-in state, no breadcrumb.

### 2. SplitScreen (40/60, drag handle)
- **Status:** Works via code, but visually almost invisible.
- **Problem:** The separator is only 1.5px wide with `bg-white/10` and no icon/grip. Most users will never discover it. No cursor feedback unless actively dragging. The ARIA attributes are good (`role="separator"`, `aria-valuenow`), but affordance is missing.
- **Evidence:** In the screenshot the divider is barely visible as a thin vertical line; no hover state was captured.

### 3. ChatPanel
- **Status:** Shell is good; content flow is broken.
- **Good:** Rounded glass panel, header with online indicator, typing area fixed at bottom, `Session Started` badge adds context.
- **Problems:**
  - Sending any message results in `Error: HTTP 500`. The fallback message is technically accurate but does not guide the user.
  - `MessageList` does not auto-scroll reliably when the message content changes because `lastContent` may be identical for consecutive errors.
  - The initial empty-state paragraph is duplicated between `MessageList` and `ResumeCanvas`.

### 4. ModeToggle (Aether / HR Coach)
- **Status:** Present and accessible (`role="tablist"`).
- **Problem:** HR Coach is a paid feature, yet the tab is always visible and clickable. Clicking it opens a ProFeatureGate overlay that blurs the content, but the active tab still appears selected, creating a mismatch between tab state and visible content. Better approach: disable the tab with a lock icon and tooltip.
- **Evidence:** Screenshot shows "HR Coach" tab selected while the panel still shows Aether content behind the paywall.

### 5. VoiceButton
- **Status:** Uses `useVoiceInput` hook; not tested end-to-end because browser mic permissions are unavailable in Playwright.
- **Problem:** Icon is from Lucide `Mic`, but `ChatInput` also relies on `material-symbols-outlined` for the send icon. Material Symbols are not loaded (see Fonts/Icons), so the send button currently shows the literal text "send" instead of an arrow icon. Voice button itself appears correctly as a mic icon.

### 6. CanvasPanel
- **Status:** Empty-state renders; resume content path is unverified.
- **Good:** SuggestionPanel has good micro-copy, skeletons, severity badges and dismiss/apply actions.
- **Problems:**
  - ProFeatureGate is shown twice in the workspace: once around `ModeToggle` and once around `SuggestionPanel`. This is repetitive.
  - `ATSScoreWidget` is positioned `absolute -top-4 -right-4` and overlaps the canvas edge; on smaller widths it may be clipped.
  - `SuggestionPanel` is `opacity-0 pointer-events-none` when no section is active, but still mounted and possibly fetching. It should be `display: none` or unmounted to avoid unnecessary API calls.

### 7. ATS Score / CircularScore / PulseRing
- **Status:** Visibly present but meaningless.
- **Problem 1:** The score is always `0%` on first load because no resume data exists. Combined with the fallback empty state, this tells the user "you have 0% ATS match" before they have any content — demotivating.
- **Problem 2:** `PulseRing` is never rendered. `ATSScoreWidget` passes `visible={isAnalyzing ?? true}`; since `isAnalyzing` is `false` initially, `?? true` makes it `true`, but the actual class `pulse-ring` is not found in DOM. The CSS keyframe `pulse-ring` may be missing or the component is conditional elsewhere.
- **Problem 3:** Color thresholds in `CircularScore` return green/violet for ≥75, yellow for 50–74, red for <50. On dark glass background the red/yellow are not accessibility-tested for contrast.
- **Evidence:** DOM check returned `pulseRing: false` and `atsScoreArc: true`.

### 8. SuggestionPanel / SuggestionCard
- **Status:** UI is polished; feature is locked.
- **Good:** Sorting by severity, count badge, refresh button, skeletons, error box.
- **Problem:** Suggestion count badge appears even when the panel is locked behind ProFeatureGate, creating a tease that may frustrate free users. The "AI Suggestions" label in the locked-state screenshot is partially clipped/overlapping.

### 9. MobileTabBar
- **Status:** Reviewed via code (Playwright viewport was desktop).
- **Good:** 4 tabs (Chat, Resume, Score, Jobs), fixed bottom with `pb-safe`, active state styling, Lucide icons.
- **Problem:** The "Score" tab maps to a placeholder (`ATS Score breakdown placeholder`) and "Jobs" tab also shows placeholder text. In a real mobile session this would feel broken. Icons are 24px which is touch-friendly; labels are readable.

### 10. Pricing Page
- **Status:** Visually strong, but functionally a stub.
- **Good:** Clear two-card layout, "Most Popular" badge, yearly/monthly toggle with "Save 20%" highlight, feature lists with checkmarks, FAQ section.
- **Problems:**
  - FAQ admits "Billing is currently in stub mode" and "payment processing is not yet enabled". For a live app this is a trust issue.
  - "Subscribe" button has no price confirmation or loading state; clicking it just updates local store.
  - No currency localization, no trial CTA, no refund policy.

### 11. Telegram Page
- **Status:** Minimal, coherent fallback.
- **Good:** Explains context, provides "Open Full Workspace" CTA, uses CSS variables for Telegram theme.
- **Problem:** The emoji 📱 is used instead of a real Telegram logo; no bot username / deep link; no instructions for how to open inside Telegram. Title says "CV Builder — Telegram" but content says "Open in Telegram".

### 12. Landing Page (/)
- **Status:** Severely underbuilt for a commercial SaaS.
- **Problems:**
  - No hero value proposition, no social proof, no feature explanation, no pricing teaser.
  - "More features soon" card is a dead end.
  - Header does not highlight current page, no footer.

## Cross-Cutting Issues

### Fonts / Icons
- **Material Symbols not loaded.** Many UI elements use `<span className="material-symbols-outlined">smart_toy</span>`, `<span>send</span>`, `<span>refresh</span>`, `<span>badge</span>`. The browser check returned `materialSymbols: false` and the screenshots show these rendered as plain text labels. This is a major visual/UX bug.
- Font families loaded: Inter, Geist, and fallbacks. The Material Symbols font face is missing from `_layout.tsx` or `globals.css`.

### Error Handling
- Any chat request fails with HTTP 500. The error message is shown as Markdown in `AgentMessage` which is fine, but:
  - There is no "Retry" action.
  - The input is cleared even on failure, so the user loses their prompt.
  - There is no indication whether the backend is down or the user lacks a subscription.

### Accessibility
- ModeToggle and MobileTabBar use correct `role="tablist"` / `role="tab"` / `aria-selected`.
- SplitScreen separator has ARIA values and keyboard support (ArrowLeft/Right).
- `ChatInput` send button uses `aria-label`; VoiceButton has `aria-label` and `title`.
- However, color contrast of muted text `#c4c7c7` on dark glass backgrounds may fail WCAG AA in some contexts.

### Performance
- `CanvasPanel` debounces ATS scoring at 2s but still fires even when the resume is empty (returns early only if both `fullName` and `jobTitle` are missing, but the effect runs and sets up a timeout each render).
- Suggestion fetch runs whenever `activeSection` changes, even if the panel is locked behind ProFeatureGate.

## Prioritized Recommendations

| Priority | Issue | Suggested Fix |
|----------|-------|---------------|
| P0 | Chat endpoint returns HTTP 500 | Fix `/api/ai/route` and add proper error boundaries |
| P0 | Material Symbols not loaded | Add `<link href="https://fonts.googleapis.com/css2?family=Material+Symbols+Outlined" rel="stylesheet">` to layout or self-host |
| P1 | SplitScreen drag handle invisible | Add 3-dot grip icon, wider hover lane (8px), stronger color on hover |
| P1 | ATS score shows 0% for empty resume | Hide score or show "—" / "Start building" until content exists |
| P1 | Voice/Send icon inconsistency | Use Lucide `Send` consistently instead of Material Symbols, or load both fonts reliably |
| P2 | HR Coach tab selectable but locked | Disable tab, add lock icon + tooltip, keep paywall on the locked panel only |
| P2 | SuggestionPanel locked but still fetches | Do not fetch until ProFeatureGate is satisfied (or unmount panel) |
| P2 | Duplicate ProFeatureGate | Consolidate upgrade prompts; avoid two "Upgrade to Pro" buttons on screen |
| P2 | Landing page is bare | Add hero, feature grid, testimonials/placeholder, footer, pricing link |
| P3 | Telegram page missing deep link | Add `@YourBot /start` link and `tg://resolve?domain=` fallback |
| P3 | Pricing stub disclaimers | Move to a "Coming soon" badge rather than inside FAQ; remove if live |
| P3 | Mobile placeholders | Implement Score/Jobs panels or hide tabs until ready |

---

## Conclusion

The interface looks modern and the component structure is sensible, but the workspace is currently non-functional for its primary use case (AI chat). Icon/font loading and backend stability must be fixed before any UI polish is meaningful. Once functional, the biggest UX wins will come from improving the drag handle affordance, removing duplicate paywalls, and clarifying the empty state for new users.
